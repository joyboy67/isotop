NAME

isotop — une OpenBSD préconfigurée, simple (et amusante ?)

DESCRIPTION

isotop est un script permettant de configurer OpenBSD pour obtenir un environnement de travail rapide, agréable et pratique. Les outils présents de base dans OpenBSD sont privilégiés. Par conséquent, certains choix ont été faits pour se concentrer sur un système simple et efficace.

FAQ

Comment me servir de ce bureau?

Le gestionnaire de fenêtres par défaut dans isotop est dwm avec quelques modifications. Sa simplicité permet de mieux se concentrer et à être plus efficace.

Avant d'installer un autre gestionnaire de fenêtres ou environnement de bureau, essayez dwm. Nous avons essayé de le rendre plus simple à utiliser qu'il ne l'est déjà, et cela demandera peut-être un temps d'adaptation.

Pour commencer :

En haut à gauche, vous voyez les numéros des espaces de travail suivi d'un symbole. Vous pouvez cliquer dessus pour en changer.

En haut à droit, une barre de status indique l'heure, la date, le volume sonore, la charge de la batterie et la chanson jouée.

Quelques raccourcis :

  • clic gauche sur la barre de status ou le fond d'écran : affiche le menu pour ouvrir une application
  • clic droit sur la barre de status ou le fond d'écran : affiche le menu de session
  • clic gauche ou scroll sur le titre d'une fenêtre : affiche la fenêtre suivante.
  • clic droit sur le titre d'une fenêtre : ferme la fenêtre.
  • ctrl-shift-Enter : ouvre un terminal
  • alt-p: affiche dmenu pour ouvrir une application

Bien sûr, ce n'est qu'un mince aperçu.

Voir dwm-isotop(1) pour plus d'informations sur les possibilités de la session isotop.

Comment installer isotop pour un autre utilisateur?

Facile, il suffit de lancer le script d'installation avec cet utilisateur. Seule la configuration utilisateur sera copiée si la dernière version d'isotop est installée.

Comment recompiler moi-même dwm, dmenu, slstatus, lsdesktop?

Ces outils sont disponibles dans le dossier isotop-files/src dispnibles après installation dans votre dossier personnel. Ils sont toujours là si vous ne les avez pas supprimés. Sinon, voir ici :

https://framagit.org/3hg/isotop/-/tree/master/isotop-files/src

Comment ajouter une imprimante?

Ouvrez un navigateur à l'adresse

http://127.0.0.1:631

Puis cliez sur l'onglet “Administration”

Comment configurer une connexion WiFi?

Lisez le man hostname.if(5) et la documentation d'OpenBSD traduite réseau sans fil Un outil pour faciliter les connexions sera publié dans une prochaine version d'isotop, lorsque le développeur aura une carte sans-fil.

Comment ajouter/supprimer des programmes

Ouvrez pkg_mgr(1) dans un terminal.

Sinon, lisez gestion des paquets.

Pourquoi mon écran semble rouge?

Puisqu'on pense à vos yeux, on lance automatiquement en début de session sct(1) Vous pouvez utiliser dsct(1) pour modifier la couleur, ou carrément retirer sctd 1>/dev/null & du fichier $HOME/.start

Pourquoi mpd est le lecteur de musique par défaut?

Il est léger et ne nécessite pas de lourdes dépendances. Par ailleurs, c'est très simple de le contrôler grâce à divers scripts. L'outil ncmpc est installé pour vous donner une interface à mpd. Par défaut, toute la musique placée dans le dossier $HOME/Musique sera disponible à la lecture.

À ce propos, voir :

  • dmpc(1)
  • mpc(1)
  • ncmpc(1)

Quelles sont les différences avec une installation OpenBSD de base?

  • unwind(8) est configuré en tant que résolveur de nom de domaine local par défaut. Ainsi, vous ne dépendez pas d'un fournisseur d'accès à internet pour réaliser cette tâche (anti-censure).

    Un fichier /var/unwind.block contient une liste de domaines connus comme malveillants ou publicitaires. Cette liste est mise à jour une fois par semaine au démarrage via le script /etc/rc.local.

    Cette liste est récupérée depuis le site de l'auteur d'isotop.

    Si vous essayez d’accéder à l’un de ces domaines (pourquoi?), Vous obtiendrez un message:

    “Le site web n'est pas disponible”

  • hotplugd(8) est configuré pour monter automatiquement votre clé USB dans /vol

    Un lien est disponible dans votre répertoire $HOME/Médias.

  • Services activés: hotplugd, cups, xenodm, unwind.
  • Désactivation de ulpt pour les imprimantes USB. Cela se passe dans /etc/rc.shutdown pour profiter de KARL au redémarrage.

    Voir ce fil

  • La configuration ntpd n'utilise pas google comme contrainte.
  • L’apparence de Xenodm est personnalisée, traduite et configurable avec un script appelé xenodm-config.
  • Des paquets supplémentaires sont installés.

    Voir https://framagit.org/3hg/isotop/-/blob/master/isotop-files/packages.txt

  • doas est configuré pour éviter la saisie d'un mot de passe pour certaines commandes.

    Voir /etc/doas.conf

  • firefox(1) est le navigateur par défaut. Il est "unveiled" par défaut: cela signifie que le navigateur ne peut lire des fichiers sur votre ordinateur sauf dans $HOME/Downloads et /tmp pour plus de sécurité.

    Il garde vos clés SSH et mots de passe inaccessibles par un éventuel pirate.

  • Alt-L est mappé vers Esc pour les utilisateurs de vi :)
  • Les traductions sont incluses: anglais et français pour l'instant.
  • Quelques scripts et outils sont inclus dans votre dossier personnel /$HOME/bin et disponible dans votre PATH . Ainsi, vous pouvez facilement les modifier ou les supprimer s'ils ne vous plaisent pas.

    Voir Quels scripts sont livrés?

  • $HOME/.aliases et $HOME/.functions sont remplis de choses (utiles?).
  • $HOME/.kshrc contient de l'auto-complétion pour OpenBSD.
  • Si un fichier $HOME/.wallpaper est trouvé, alors il est définit comme fond d'écran. Copiez le fond d'écran de votre choix à sa place.
  • Le navigateur de fichiers est xfe

    Il est à la fois léger et suffisamment configurable pour permettre l'accès à des actions personnalisées via clic-droit.

  • Le pointeur de la souris est caché grâce à xbanish(1)
  • Le thème de couleur "nord" est installé pour les applications X. Voir. https://www.nordtheme.com/

Pourquoi ce nom?

Au début, isotop était image iso contenant tous les paquetages et configuration pour l'installation. Maintenant, c'est juste un script pour configurer OpenBSD.

Comme un isotope c'est une variante d'OpenBSD avec quelques éléments supplémentaires, mais toujours OpenBSD.

Pourquoi c'est sombre?

Parce que le développeur aime bien, et ça économise l'énergie de ne pas allumer les pixels.

Puis-je installer un autre environnement de bureau?

Bien sûr.

Gnome, kde, lxde, xfce et d'autres sont disponibles pour OpenBSD via les ports.

Quels processus sont démarrés lorsque j'ouvre une session?

Consultez le fichier $HOME/.start

Quels scripts sont inclus?

  • bdmenu: permet de lancer une commande via dmenu avec historique et possibilité d'exécuter une commande dans un terminal en terminant l'entrée par “”!
  • checkbatt : ce script est lancé périodiquement via crontab(5) et permet d'afficher une alerte si la batterie devient faible.
  • dalarm: spécifiez une alerte, puis un nombre de minutes. Une notification affichera l'alerte après le délai.
  • ddesktop : affiche dans un dmenu une liste des applications disponibles sur le système après avoir listé les fichiers Exemple: top!

    Ce menu est disponible en faisant un clic-gauche sur la barre de status ou le fond d'écran.

  • dfm: un lanceur de fichiers utilisant dmenu.
  • dloadbmk: ouvre une page web listée dans $HOME/.bookmarks
  • dman : permet d'ouvrir rapidement une page man (merci eol)
  • dmpc: choix d'une chanson à jouer dans mpd
  • dsch: recherche sur le web. Le moteur de recherche par défaut est duckduckgo si vous ne spécifiez pas de recherche moteur.

    Exemples :

    moteur par défaut: OpenBSD, chiffrement complet du disque

    recherche sur openbsd misc list: obsdmisc firefox crash

  • dsct : configurer la couleur de l'écran
  • dshot : capture d'écran avec possibilité d'upload et copie dans le presse papier
  • dsshmenu : ouvre une session ssh.
  • dyt: télécharger une vidéo avec youtube-dl(1)

    L'URL sélectionnée est collée par défaut.

  • gdoas: ouvre xterm pour taper le mot de passe doas utilisé pour d’autres scripts
  • imgmod: modifier une image en utilisant gm(1)

    Les modifications supportées sont.

    • convertir en jpg
    • convertir en png
    • convertir en gif
    • redimensionner
    • tourner à gauche ou à droite
  • imgopti: optimise une image pour réduire son poids.
  • networkcheck: vérifie l'accès à Internet
  • openbsdupgrade: mise à niveau vers les dernières modifications de -stable avec syspatch(8) fw_update(1) pkg_add(1)
  • pixcol: affiche le code html de la couleur du pixer situé sous le pointeur.
  • pixup: téléverse une image sur pix.toile-libre.org
  • scrot : réalise une capture d'écran.
  • send_dmesg.sh: envoyez votre dmesg aux développeurs OpenBSD pour aider au support matériel.
  • sessionmenuisotop: Ouvre une fenêtre pour éteindre/redémarrer/quitter la session...

    Ce menu est disponible en faisant un clic-droit sur la barre de status ou le fond d'écran.

  • setwall: définit le fond d'écran.
  • term: ouvre un terminal. PAr défaut, xterm est utilisé.

À propos de dmenu

Les outils inclus utilisent dmenu patché pour pouvoir aussi l'utiliser avec la souris.

C'est un menu affiché en haut de votre écran. Écrivez ce dont vous avez besoin, ou une partie seulement, puis appuyez sur Entrée pour sélectionner ou bien cliquez dessus la lignee désirée. Un clic droit ou bien la touche echap ferment le menu.

Retour ou clic gauche: sélectionner
Esc ou clic droit: quitter le menu.
ctrl-y: coller la sélection

Voir site web de dmenu
